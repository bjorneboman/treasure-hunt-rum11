Studieguide Vecka 36: Uppstart, utvecklingsmiljö och agila metoder Övergripande mål för veckan: • Förstå kursens upplägg, roller och förväntningar. • Få en grundläggande förståelse för Scrum och agila arbetssätt. • Kunna skapa och enkelt dokumentera GitHub-projekt. • Installera och konfigurera utvecklingsmiljö (VS Code, terminal, Git). Veckans Learning Targets (Utifrån kursmålen) Kunskaper • K5: Förklara grunderna i Git och hur det används för versionshantering. • K9: Beskriva de grundläggande principerna inom agila metoder, inklusive iterativ utveckling, feedback-loopar och teamarbete. • K10: Beskriva vikten av dokumentation i mjukvaruutveckling och identifiera grundläggande typer av dokumentation, såsom kodkommentarer och README-filer. Färdigheter • F3: Använda Git för att skapa och hantera repositories. • F4: Skriva tydliga kodkommentarer och en enkel README-fil som förklarar syftet och användningen av ett grundläggande programmeringsprojekt. Onsdag 3/9: Workshop - VS Code & arbetsmiljö. Tid: 13.00 – 16.00 Mål: • Ha installerat VS Code och utvalda extensions. • Skapa och arbeta i ett projekt kopplat till GitHub. • Förstå hur man samarbetar i en gemensam arbetsmiljö. Torsdag 4/9: Workshop - Terminal, CLI och Git Tid: 09.00 – 12.00 Mål: • Använda terminalkommandon för navigering och filhantering. • Visa att du kan initiera, committa och pusha till GitHub. Förberedelse Läsning: • Git Basics – W3Schools • Command Line Crash Course (Från 8:40min - 11.00min är GIT-delen) • Command Line for Beginners (Ubuntu) Praktiska övningar: 1. Se till att du har installerat Git på din dator. (t.ex. mha. ChatGPT). 2. Skapa en ny mapp i VS Code och klona ett av dina GitHub-repon via terminalen. 3. Träna att navigera mellan mappar och skapa filer via terminalen. 4. Initiera fem Git-repos och gör minst tre commits från varje via terminalen. 5. Pusha alla projekt till GitHub. 6. Läs på och lär dig använda dessa git-kommandon.
